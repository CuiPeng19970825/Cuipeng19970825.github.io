一、线性结构—双向链表

发表于 2019-08-23 | 分类于 数据结构与算法

双项链表学习整理

双向链表

单向链表的缺点在于每个节点只能知道自己与下一个节点，无法寻找到上一个节点。而双向链表可以

解决这个问题，通过在节点内部添加一个pre指针，指向它的前一节点。

优点：支持向前遍历，也支持向后遍历；

缺点：在对链表进行增删的时候需要对2个以上指针进行操作。

基类的定义

  • 定义一个节点类DoubleNode

  • insert()方法，插入一个节点

  • pre()方法，获取上一个节点

  • next()方法，获取下一个节点

  • getData()方法，获取节点中的数据

代码实现

package demoDoubleNode;
       public class DoubleNode {
           //上一个节点
           private DoubleNode pre;
           //下一个节点
           private DoubleNode next;
           //数据
           private int data;

           public DoubleNode(int data){
               pre=this;
               next=this;
               this.data=data;
           }

           //插入一个节点
           public void insert(DoubleNode newNode){
               //当前节点
               DoubleNode currentNode=this;
               //当前节点的下一个节点之后作为下下一个节点
               DoubleNode NextNode=this.next;
               //令将当前节点的下一个节点指向新节点
               currentNode.next=newNode;
               //将新节点的前一个节点指向当前节点
               newNode.pre=currentNode;
               //将新节点的下一个节点指向下下一个节点
               newNode.next=NextNode;
               //将下下一个节点的前一个节点指向新节点
               NextNode.pre=newNode;
           }

           //获取上一个节点
           public DoubleNode pre(){
               return this.pre;
           }

           //获取下一个节点
           public DoubleNode next(){
               return this.next;
           }

           //获取数据
           public int getData(){
               return this.data;
           }

       }

测试代码

public class TestDoubleNode {
       public static void main(String[] args) {
           DoubleNode n1=new DoubleNode(1);

           DoubleNode n4=new DoubleNode(4);
           System.out.println(n1.pre().getData());
           System.out.println(n1.next().getData());
           n1.insert(n2);

           n3.insert(n4);
           System.out.println(n1.pre().getData());
           System.out.println(n4.next().getData());

       }
   }

一、线性结构—循环链表

发表于 2019-08-23 | 分类于 数据结构与算法

循环链表学习整理

循环链表

是在单链表的基础上进行的改进，最后一个结点的链域值不为NULL，而是指向头结点。

基类的定义

  • 定义一个节点类LoopNode

  • Next()方法，获取下一个节点

  • getData()方法，获取节点中的数据

  • remove()方法，删除下一个节点

  • insert()方法，插入一个节点作为当前节点的下一个节点

代码实现

package demoLoopNode.copy;
public class LoopNode {
           //数据
           private int data;

           //下一个节点
           private LoopNode next=this;

           //构造方法
           public LoopNode(int data){
               this.data=data;
           }

           //获取下一个节点
           public LoopNode Next(){
               return this.next;
           }

           //获取节点中的数据
           public int getData(){
               return this.data;
           }


           //删除下一个节点
           public void remove(){
               //取出下一个节点
               LoopNode newNext=next.next;
               //把下下个节点设置为当前节点的下一个节点
               this.next=newNext;

           }

           //插入一个节点作为当前节点的下一个节点
           public void insert(LoopNode newNode){
               LoopNode currentNode=this;
               newNode.next=currentNode.next;
               currentNode.next=newNode;
           }

       }

测试代码

一、线性结构—单链表

发表于 2019-08-23 | 分类于 数据结构与算法

单链表学习整理

单链表

通过一个链子把多个结点（元素）连接起来，由数据和地址组成的一个元素，节点本身必须有一个

地址值（就是下一个元素的地址值）链表的每个结点中只包含一个指针域，叫做单链表（即构成链

表的每个节点只有一个指向直接后继结点的指针，且尾节点的指针为null）。

优点：

  1. 链表是很常用的一种数据结构，不需要初始化容量，可以任意加减元素；

  2. 添加或者删除元素时只需要改变前后两个元素结点的指针域指向地址即可，所以添加，

删除很快；

缺点：

  1. 因为含有大量的指针域，占用空间较大；

  2. 查找元素需要遍历链表来查找，非常耗时。

适用场景：

数据量较小，需要频繁增加，删除操作的场景。

基类的定义

  • 定义一个节点类Node

  • append()方法,为当前节点追加节点

  • Next()方法，获取下一个节点

  • getData()方法，获取节点中的数据

  • isLast()方法，判断当前节点是否是最后一个节点

  • remove()方法，删除下一个节点

  • show()方法，显示所有节点信息

  • insert()方法，插入一个节点

代码实现

package demoNode;
public class Node {
       //数据
       private int data;
       //下一个节点
       private Node next;

       //构造方法
       public Node(int data){
           this.data=data;
       }

       //为当前节点追加节点
       public Node append(Node node){
           //当前节点
           Node currentNode=this;
           //循环查找
           while(true){
               //取出下一个节点
               Node nextNode=currentNode.next;
               //如果下一个节点为空，当前节点已经是最后一个节点
               if(nextNode==null){
                   break;
               }
               //赋值给当前节点
               currentNode=nextNode;
           }
           //把需要追加的节点追加到找到的当前节点的下一个节点
           currentNode.next=node;
           return this;
       }

       //获取下一个节点
       public Node Next(){
           return this.next;
       }

       //获取节点中的数据
       public int getData(){
           return this.data;
       }

       //判断当前节点是否是最后一个节点
       public boolean isLast(){
           return next==null;
       }

       //删除下一个节点
       public void remove(){
           //取出下一个节点
           Node newNext=next.next;
           //把下下个节点设置为当前节点的下一个节点
           this.next=newNext;

       }

       //显示所有节点信息
       public void show(){
           Node currentNode=this;
           while(true){
               System.out.print(currentNode.data+" ");
               //取出下一个节点
               currentNode=currentNode.next;
               if(currentNode==null){
                   break;
               }
           }
           System.out.println();
       }

       //插入一个节点
       public void insert(Node newNode){
           Node currentNode=this;
           newNode.next=currentNode.next;
           currentNode.next=newNode;
       }

   }

测试代码

 package demoNode;
 public class TestNode {
    public static void main(String[] args) {
        //创建节点对象
        Node n1=new Node(1);
        Node n2=new Node(2);
        Node n3=new Node(3);
        //用追加节点方法将相应的节点连成单链表
        n1.append(n2).append(n3).append(new Node(4));
        //取出下一个节点中的数据
        System.out.println(n1.Next().Next().getData());
        System.out.println(n1.isLast());
        //显示所有节点信息
        n1.show();
        //删除某个节点的下一个节点
        n1.remove();
        n1.show();
        //插入节点
        n1.insert(n2);
        //追加新节点
        n1.append(new Node(5));
        n1.show();
        System.out.println(n2.getData());
    }
}

一、线性结构—队列

队列学习整理

队列

队列(queue)是一种特殊的线性表 ，它只允许在表的前端（front）进行删除操作，而在表的后端

（rear）进行插入操作。进行插入操作的端称为队尾，进行删除操作的端称为队头。队列是按照“先

进先出”或“后进后出”的原则组织数据的。队列中没有元素时，称为空队列。

优点：提供先进先出的存取方式

缺点：存取其他项都很慢

基类的定义

  • 定义一个队列类Queue

  • add()方法，入队

  • poll()方法，出队

  • isEmpty()方法，判断队列是否为空

代码实现

package demoQueue;
public class Queue {
       int[] arr;
       public Queue(){
           arr=new int[0];
       }

       //入队
       public void add(int element){
           // 新创建一个数组
           int[] newarr = new int[arr.length + 1];
           // 将原数组中的元素复制到新数组中
           for (int i = 0; i < arr.length; i++) {
               newarr[i] = arr[i];
           }
           // 将新增加的元素放入新数组中
           newarr[arr.length] = element;
           // 新数组替换原数组
           arr = newarr;
       }

       //出队
       public int poll(){
           int element=arr[0];
           //新创建一个数组
           int[] newarr=new int[arr.length-1];
           //将未出栈的元素复制到新数组中
           for(int i=0;i<newarr.length;i++){
               newarr[i]=arr[i+1];
           }
           //新数组替换原数组
           arr=newarr;
           //返回出队的元素
           return element;
       }

       //判断队列是否为空
       public boolean isEmpty(){
           return arr.length==0;
       }
   }

测试代码

一、线性结构—栈

发表于 2019-08-23 | 分类于 数据结构与算法

栈学习整理

栈

栈是一种只允许在一端进行插入或删除的线性表，也就是说先进后出。栈的操作端通常被称为栈顶，

另一端被称为栈底，栈的插入操作称为压栈（push），栈删除操作称为出栈（pop）。压栈是把新元

素放到栈顶元素的上面，使之成为新的栈顶元素；出栈则是把栈顶元素删除掉，使其相邻的元素成

为新的栈顶元素。

优点：提供后进先出的存取方式

缺点：存取其他项很慢

基类的定义

  • 定义一个栈类Stack

  • push()方法，压入元素

  • pop()方法，取出栈顶元素

  • size()方法，显示栈的大小

  • show()方法，查看栈顶元素

  • isEmpty()方法，判断栈是否为空

代码实现

 package demoStack;
 public class Stack {
        private int[] arr;
        public Stack(){
            arr=new int[0];
        }

        //压入元素
        public void push(int element){
            // 新创建一个数组
            int[] newarr = new int[arr.length + 1];
            // 将原数组中的元素复制到新数组中
            for (int i = 0; i < arr.length; i++) {
                newarr[i] = arr[i];
            }
            // 将新增加的元素放入新数组中
            newarr[arr.length] = element;
            // 新数组替换原数组
            arr = newarr;
        }

        //取出栈顶元素
        public void pop(){
            //新创建一个数组
            int[] newarr=new int[arr.length-1];
            //将原数组中的元素复制到新数组中
            for(int i=0;i<newarr.length;i++){
                newarr[i]=arr[i];
            }
            //新数组替换原数组
            arr=newarr; 
        }

        //显示栈的大小
        public int size(){
            return arr.length;
        }

        //查看栈顶元素
        public void show(){
            System.out.println(arr[arr.length-1]);
        }

        //判断栈是否为空
        public boolean isEmpty(){
            return arr.length==0;
        }
}

一、线性结构—数组

发表于 2019-08-22 | 更新于 2019-08-23 | 分类于 数据结构与算法

数组学习整理

数组

数据结构中最基本的一个结构就是线性结构，而线性结构又分为连续存储结构和离散存储结构，

所谓的连续存储结构其实就是数组。

优点 ：

  1. 按照索引查询元素速度快

  2. 按照索引遍历数组方便

缺点：

  1. 数组的大小固定后就不能扩容了

  2. 数组只能存储一种类型的数据

  3. 添加，删除的操作慢，因为要移动其他的元素

适用场景：

频繁查询，对存储空间要求不大，很少增加和删除的情况

基类的定义

  • 定义一个数组类Arr

  • size()方法，获取数组长度

  • add()方法，向数组末尾增加一个元素

  • show()方法，显示数组中的所有元素

  • delete()方法，删除数组中某个位置的元素

  • insert()方法，向数组中插入一个元素

  • get()方法，获取数组中目标位置的元素

  • set()方法，替换数组中目标位置的元素值

  • search()线性查找方法

  • BinarySearch()二分法查找，条件：查找的数组有序

代码实现

package demoArray;
import java.util.Arrays;
public class Arr {
    private int[] arr;

    public Arr (){
        arr=new int[0];

    }

    //获取数组长度
    public int size(){
        return arr.length;
    }

    //向数组末尾增加一个元素
    public void add(int element){
        //新创建一个数组
        int[] newarr=new int[arr.length+1];
        //将原数组中的元素复制到新数组中
        for(int i=0;i<arr.length;i++){
            newarr[i]=arr[i];
        }
        //将新增加的元素放入新数组中
        newarr[arr.length]=element;
        //新数组替换原数组
        arr=newarr;
    }

    //显示数组中的所有元素
    public void show(){
        System.out.println(Arrays.toString(arr));
    }

    //删除数组中某个位置的元素
    public void delete(int index){
        //创建一个新数组
        int[] newarr=new int[arr.length-1];
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        for(int i=0;i<newarr.length;i++){
            //将删除元素前的元素复制到新数组中
            if(i<index){
            newarr[i]=arr[i];
            //将删除元素后的元素复制到新数组中
            }else{
                newarr[i]=arr[i+1];
            }
        }
        //新数组替换原数组
        arr=newarr;
    }

    //向数组中插入一个元素
    public void insert(int index,int element){
        //创建一个新数组
        int[] newarr=new int[arr.length+1];
        //判断下标是否越界
        if(index<0||index>newarr.length-1){
            throw new RuntimeException("下标越界");
        }
        for(int i=0;i<arr.length;i++){
            //将插入元素前的元素复制到新数组
            if(i<index){
                newarr[i]=arr[i];
            //将插入元素后的元素复制到新数组
            }else{
                newarr[i+1]=arr[i];
            }
        }
        //将插入元素放入到新数组中
        newarr[index]=element;
        //新数组替换原数组
        arr=newarr;
    }

    //获取数组中目标位置元素
    public int get(int index){
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        return arr[index];
    }

    //替换数组中目标位置元素的值
    public void set(int index,int element){
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        arr[index]=element;
    }

    //线性查找
    public int search(int target) {
        for(int i=0;i<arr.length;i++){
            if(arr[i]==target){
                return i;
            }
        }
        return -1;
    }

    //二分法查找
    public int BinarySearch(int target){
        // 记录开始位置
        int begin = 0;
        // 记录结束位置
        int end = arr.length - 1;
        // 记录中间的位置
        int mid = (begin + end) / 2;
        // 循环查找
        while (true) {
            //判断什么情况下没有目标元素
            if(begin>=end){
                return -1;
            }
            // 判断中间的这个元素是否是目标元素
            if (arr[mid] == target) {
                return mid;
            } else {
                if (arr[mid] > target) {
                    end = mid - 1;
                } else {
                    begin = mid + 1;
                }
                mid = (begin + end) / 2;
            }
        }
    }
 }

TCP/IP

发表于 2019-08-16 | 更新于 2019-08-22 | 分类于 网络

一、介绍：

TCP/IP是用于因特网的通信协议，计算机通信协议是对那些计算机必须遵守以便彼此通

信的规则的描述。

二、定义：

  1. TCP/IP是供已连接因特网的计算机进行通信的通信协议；

  2. TCP/IP指传输控制协议/网际协议（Transmission Control Protocol/Internet

    Protocol）；

  3. TCP/IP定义了电子设备（比如计算机）如何连入因特网，以及数据如何在他们之间传

    输的标准。

三、作用：

在TCP/IP中包含一系列用于处理数据通信的协议。

  • TCP(传输控制协议)-应用程序之间通信；

  • UDP(用户数据报协议)-应用程序之间的简单通信；

  • IP(网际协议)-计算机之间的通信；

  • ICMP(因特网消息控制协议)-针对错误和状态；

  • DHCP(动态主机配置协议)-针对动态寻址。

TCP使用固定的连接

  1. 请求必须被送到一个确切的地址；

  2. 两个应用程序之间建立一个全双工通信，并且占用两个计算机之间的通信线路，

    直到它被一方或双方关闭为止；

UDP和TCP很相似，但更简单，同时可靠性低于TCP。

IP是无连接的

  1. 不占用两个正在通信的计算机之间的通信线路；

  2. 消息(或其他数据)被分割为小的独立的包通过因特网在计算机之间传送；

  3. IP负责将每个包路由至它的目的地；

  4. 当一个IP包从一台计算机被发送，它会到达一个IP路由器；

  5. IP路由器负责将这个包路由至它的目的地，直接地或是通过其他地路由器；

TCP/IP

  1. TCP/IP意味着TCP和IP在一起协同工作；

  2. TCP负责应用软件(比如浏览器)和网络软件之间地通信；

  3. IP负责计算机之间的通信；

  4. TCP负责将数据分割并装入IP包，然后在他们到达的时候重新组合他们；

  5. IP负责将包发送至接收者。

git学习

发表于 2019-08-13 | 更新于 2019-08-15 | 分类于 工具

前言：git的学习记录，学习参考网站Git教程-廖雪峰的官方网站

一、

git定义：分布式版本控制系统

git作用：记录、管理文件的每一次改动，支持协作编辑。

  1. 版本控制系统：记录软件更新内容的系统

  2. 集中式：版本库是集中存放在中央服务器中的，工作时需要从中央服务器中取得最新的版本，更

    新版本内容后再把最新的版本提交到中央服务器，举例：中央服务器就好比是一个图书馆，你要

    改一本书，必须先从图书馆借出来，然后回到家自己改，改完了，再放回图书馆。

  3. 分布式：分布式版本控制系统没有“中央服务器”，每个人的电脑上都是一个完整的版本库，协作

    时只需将更新的内容推送给对方即可。

  4. 集中式vs分布式：集中式版本控制系统最大的毛病就是必须联网才能工作，而分布式版本控制系

    统没有中央服务器，每个人电脑里都有完整的版本库，不需要联网，同时提高了安全性问题，当

    某一人电脑坏了的时候只需要从其他人电脑里把版本库复制一份即可，而集中式版本控制系统若

    中央服务器出问题会导致所有人都无法工作。

二、

同类工具：

  1. CVS最早的开源而且免费的集中式版本控制系统，直到现在还有不少人在用。由于CVS自身设

    计的问题，会造成提交文件不完整，版本库莫名其妙损坏的情况。同样是开源而且免费的SVN

    修正了CVS的一些稳定性问题，是目前用得最多的集中式版本库控制系统。

  2. 除了免费的外，还有收费的集中式版本控制系统，比如IBM的ClearCase（以前是Rational公司

    的，被IBM收购了），特点是安装比Windows还大，运行比蜗牛还慢。微软自己也有一个集中式版

    本控制系统叫VSS，集成在Visual Studio中。

  3. 分布式版本控制系统除了Git以及促使Git诞生的BitKeeper外，还有类似Git的Mercurial和Ba

    zaar等。这些分布式版本控制系统各有特点，但最快、最简单也最流行的依然是Git！

三、

安装

  1. 安装教程连接

    https://git-scm.com/book/zh/v2/%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git

语法和用法：

  1. 创建版本库

    mkdir blogimages 版本库又名仓库，英文名respository,blogimages为仓库名

    cd blogimages 切换到blogimages仓库

    pwd 显示当前目录位置

  2. 初始化仓库

    git init 通过该命令把这个目录变成git可以管理的仓库

  3. 将更新的内容放到该仓库中 这里在该仓库中添加了一个img文件夹

  4. 将文件添加到仓库

    git add img 该命令可反复多次使用，添加多个文件

  5. 将文件提交到仓库

    git commit -m “add two images” -m后面输入的是本次提交的说明

  6. 查看仓库当前的状态

    git status 该命令可以让我们时刻掌握仓库当的状态

  7. 查看修改前后的差别

    git diff 查看修改前后的difference

  8. 查看提交历史

    git log git log命令显示从最近到最远的提交日志

  9. 从当前版本回退到上一个版本

    git reset -hard Head^

    Head表示当前版本，Head^表示上一个版本，Head^^表示上上一个版本，Head~100表示往上100个版本

  1. 查看命令历史

    git reflog git提供了一个命令git reflog用来记录你的每一次命令

  2. 工作区和暂存区

    • 在你电脑里能看到的目录，即上面所建的blogimages为工作区

    • 工作区中的.git不算工作区，而是git的版本库

    • 版本库中的stage(或者叫index)为暂存区，master为git自动给我们创建的第一个分支

    • 前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的：

      第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区；

      第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。

  3. 查看工作区和版本库里最新版本的区别

    git diff Head – img

  4. 丢弃工作区的修改

    git checkout – img

注释：

一种是img自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的
状态；
一种是img已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后
的状态,总之，就是让这个文件回到最近一次git commit或git add时的状态。

  1. 把暂存区的修改撤销掉，重新放回工作区

    git reset Head img

    由此可知道git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区

  2. 删除文件

    rm 文件名

    git rm 文件名

注释:

用 git rm 来删除文件，同时还会将这个删除操作记录下来；
用 rm 来删除文件，仅仅是删除了物理文件，没有将其从 git 的记录中剔除。

添加远程库

  1. 将本地的仓库与GitHub上的仓库相关联

    git remote add origin git@github.com:github用户名/GitHub仓库名.git

    origin就是远程库的名字，这是git默认的叫法

  2. 把本地库的所有内容推送到远程库上

    git push -u origin master

注释：

加上-u参数git不仅会把本地的master分支内容推送到远程的master分支，而且还会
把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以
简化命令。推送成功之后可以在GitHub页面中看到远程仓库的内容已和本地一模一样。 
从现在起，可以通过git push origin master命令将master分支的最新修改推送至
github。

从远程库克隆

  1. 克隆一个本地库

    git clone git@github.com:GitHub用户名/GitHub仓库名.git

    Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快。

分支管理

  1. 创建+切换分支

    git checkout -b dev -b参数表示创建并切换,dev为新创建的分支

  2. 查看分支

    git branch 该命令会列出所有分支，当前分支前面会标有一个

  3. 切换回master分支

    git checkout master

4.把dev分支的成果合并到master分支上

git merge dev 该命令用于合并指定分支到当前分支

  1. 删除dev分支

    git branch -d dev

  2. 查看分支合并图

    git log –graph

  3. 普通模式合并

    git merge –no-ff -m “提交说明” dev

注释：

合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看
出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

  1. 储藏当前现场，等到恢复现场后继续工作

    git stash

  2. 查看储藏的工作现场

    git stash list

  3. 恢复工作现场

    • git stash apply 恢复后stash内容并不删除，需要用git stash drop来删除

    • git stash pop 恢复的同时把stash内容也删了

    • git stash apply stash@{0} 可以多次stash，恢复的时候先查看再恢复指定现场

  4. 强行删除一个没有被合并的分支

    git branch -D <分支名>

  5. 查看远程库的信息

    • git remote

    • git remote -v 显示更详细的信息

  6. 多人协作分支管理总结

    查看远程库信息，使用git remote -v；

    本地新建的分支如果不推送到远程，对其他人就是不可见的；

    从本地推送分支，使用git push origin branch-name，如果推送失败，先用git

    pull抓取远程的新提交；

    在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/

    branch-name，本地和远程分支的名称最好一致；

    建立本地分支和远程分支的关联，使用git branch –set-upstream branch-name origin/

    branch-name；

    从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

  7. 把本地未push的分叉提交历史整理成直线

    git rebase

注释:

rebase的目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方
对比。

标签管理

  1. 创建一个新标签

    git tag

    创建带有说明的标签

    git tag -a v0.1 -m”version 0.1 released” 1094adb

    用-a指定标签名，用-m指定说明文字，1094adb为提交号

  2. 查看所有标签

    git tag

  3. 查看标签详细信息

    git show

  4. 删除标签

    git tag -d v0.1

    创建的标签都存储在本地，不会自动推送到远程，打错的标签可以在本地安全删除

  5. 推送某个标签到远程

    git push origin

    一次性推送全部尚未推送到远程的本地标签

    git push origin -tags

  6. 删除远程标签

    git tag -d v0.1 先从本地删除

    git push origin :refs/tags/v0.1 然后从远程删除

  1. 配置别名

    git config –global alias.st status 用st代替单词status
