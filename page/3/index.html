git学习

发表于 2019-04-30 | 更新于 2019-08-23 | 分类于 工具

前言：git的学习记录，学习参考网站Git教程-廖雪峰的官方网站

一、

git定义：分布式版本控制系统

git作用：记录、管理文件的每一次改动，支持协作编辑。

  1. 版本控制系统：记录软件更新内容的系统

  2. 集中式：版本库是集中存放在中央服务器中的，工作时需要从中央服务器中取得最新的版本，更

    新版本内容后再把最新的版本提交到中央服务器，举例：中央服务器就好比是一个图书馆，你要

    改一本书，必须先从图书馆借出来，然后回到家自己改，改完了，再放回图书馆。

  3. 分布式：分布式版本控制系统没有“中央服务器”，每个人的电脑上都是一个完整的版本库，协作

    时只需将更新的内容推送给对方即可。

  4. 集中式vs分布式：集中式版本控制系统最大的毛病就是必须联网才能工作，而分布式版本控制系

    统没有中央服务器，每个人电脑里都有完整的版本库，不需要联网，同时提高了安全性问题，当

    某一人电脑坏了的时候只需要从其他人电脑里把版本库复制一份即可，而集中式版本控制系统若

    中央服务器出问题会导致所有人都无法工作。

二、

同类工具：

  1. CVS最早的开源而且免费的集中式版本控制系统，直到现在还有不少人在用。由于CVS自身设

    计的问题，会造成提交文件不完整，版本库莫名其妙损坏的情况。同样是开源而且免费的SVN

    修正了CVS的一些稳定性问题，是目前用得最多的集中式版本库控制系统。

  2. 除了免费的外，还有收费的集中式版本控制系统，比如IBM的ClearCase（以前是Rational公司

    的，被IBM收购了），特点是安装比Windows还大，运行比蜗牛还慢。微软自己也有一个集中式版

    本控制系统叫VSS，集成在Visual Studio中。

  3. 分布式版本控制系统除了Git以及促使Git诞生的BitKeeper外，还有类似Git的Mercurial和Ba

    zaar等。这些分布式版本控制系统各有特点，但最快、最简单也最流行的依然是Git！

三、

安装

  1. 安装教程连接

    https://git-scm.com/book/zh/v2/%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git

语法和用法：

  1. 创建版本库

    mkdir blogimages 版本库又名仓库，英文名respository,blogimages为仓库名

    cd blogimages 切换到blogimages仓库

    pwd 显示当前目录位置

  2. 初始化仓库

    git init 通过该命令把这个目录变成git可以管理的仓库

  3. 将更新的内容放到该仓库中 这里在该仓库中添加了一个img文件夹

  4. 将文件添加到仓库

    git add img 该命令可反复多次使用，添加多个文件

  5. 将文件提交到仓库

    git commit -m “add two images” -m后面输入的是本次提交的说明

  6. 查看仓库当前的状态

    git status 该命令可以让我们时刻掌握仓库当的状态

  7. 查看修改前后的差别

    git diff 查看修改前后的difference

  8. 查看提交历史

    git log git log命令显示从最近到最远的提交日志

  9. 从当前版本回退到上一个版本

    git reset -hard Head^

    Head表示当前版本，Head^表示上一个版本，Head^^表示上上一个版本，Head~100表示往上100个版本

  1. 查看命令历史

    git reflog git提供了一个命令git reflog用来记录你的每一次命令

  2. 工作区和暂存区

    • 在你电脑里能看到的目录，即上面所建的blogimages为工作区

    • 工作区中的.git不算工作区，而是git的版本库

    • 版本库中的stage(或者叫index)为暂存区，master为git自动给我们创建的第一个分支

    • 前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的：

      第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区；

      第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。

  3. 查看工作区和版本库里最新版本的区别

    git diff Head – img

  4. 丢弃工作区的修改

    git checkout – img

注释：

一种是img自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的
状态；
一种是img已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后
的状态,总之，就是让这个文件回到最近一次git commit或git add时的状态。

  1. 把暂存区的修改撤销掉，重新放回工作区

    git reset Head img

    由此可知道git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区

  2. 删除文件

    rm 文件名

    git rm 文件名

注释:

用 git rm 来删除文件，同时还会将这个删除操作记录下来；
用 rm 来删除文件，仅仅是删除了物理文件，没有将其从 git 的记录中剔除。

添加远程库

  1. 将本地的仓库与GitHub上的仓库相关联

    git remote add origin git@github.com:github用户名/GitHub仓库名.git

    origin就是远程库的名字，这是git默认的叫法

  2. 把本地库的所有内容推送到远程库上

    git push -u origin master

注释：

加上-u参数git不仅会把本地的master分支内容推送到远程的master分支，而且还会
把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以
简化命令。推送成功之后可以在GitHub页面中看到远程仓库的内容已和本地一模一样。 
从现在起，可以通过git push origin master命令将master分支的最新修改推送至
github。

从远程库克隆

  1. 克隆一个本地库

    git clone git@github.com:GitHub用户名/GitHub仓库名.git

    Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快。

分支管理

  1. 创建+切换分支

    git checkout -b dev -b参数表示创建并切换,dev为新创建的分支

  2. 查看分支

    git branch 该命令会列出所有分支，当前分支前面会标有一个

  3. 切换回master分支

    git checkout master

4.把dev分支的成果合并到master分支上

git merge dev 该命令用于合并指定分支到当前分支

  1. 删除dev分支

    git branch -d dev

  2. 查看分支合并图

    git log –graph

  3. 普通模式合并

    git merge –no-ff -m “提交说明” dev

注释：

合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看
出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

  1. 储藏当前现场，等到恢复现场后继续工作

    git stash

  2. 查看储藏的工作现场

    git stash list

  3. 恢复工作现场

    • git stash apply 恢复后stash内容并不删除，需要用git stash drop来删除

    • git stash pop 恢复的同时把stash内容也删了

    • git stash apply stash@{0} 可以多次stash，恢复的时候先查看再恢复指定现场

  4. 强行删除一个没有被合并的分支

    git branch -D <分支名>

  5. 查看远程库的信息

    • git remote

    • git remote -v 显示更详细的信息

  6. 多人协作分支管理总结

    查看远程库信息，使用git remote -v；

    本地新建的分支如果不推送到远程，对其他人就是不可见的；

    从本地推送分支，使用git push origin branch-name，如果推送失败，先用git

    pull抓取远程的新提交；

    在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/

    branch-name，本地和远程分支的名称最好一致；

    建立本地分支和远程分支的关联，使用git branch –set-upstream branch-name origin/

    branch-name；

    从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

  7. 把本地未push的分叉提交历史整理成直线

    git rebase

注释:

rebase的目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方
对比。

标签管理

  1. 创建一个新标签

    git tag

    创建带有说明的标签

    git tag -a v0.1 -m”version 0.1 released” 1094adb

    用-a指定标签名，用-m指定说明文字，1094adb为提交号

  2. 查看所有标签

    git tag

  3. 查看标签详细信息

    git show

  4. 删除标签

    git tag -d v0.1

    创建的标签都存储在本地，不会自动推送到远程，打错的标签可以在本地安全删除

  5. 推送某个标签到远程

    git push origin

    一次性推送全部尚未推送到远程的本地标签

    git push origin -tags

  6. 删除远程标签

    git tag -d v0.1 先从本地删除

    git push origin :refs/tags/v0.1 然后从远程删除

自定义git

  1. 配置别名

    git config –global alias.st status 用st代替单词status

Markdown基础语法简介

前言:

程序员写博客的专用文本标记语言markdown，让你更加极客范！

这里是本人自己初学Markdown总结的一些基本语法并写了这篇文章,学习者可以在掌握

这些基本语法后进一步学习更多Markdown写文本时的语法，初学者会遇到很多小问题,

当你自己不断摸索学习并熟练使用Markdown后你会发现使用Markdown会给你带来很多

方便。

简介:

Markdown是一种用来文本处理的轻量级「标记语言」,它用简洁的语法代替排版,而无需

像Microsoft的Word一样需要花费大量的时间进行排版、字体设置。它使我们专心于码字，

用「标记」语法，来代替常见的排版格式。Markdown不止可以处理文本，使得文字更美观,

还支持图像、表格等的插入,大大方便了我们的写作。例如此文从内容到格式，甚至插图,

一个键盘可以搞定了，无需鼠标！

Markdown特点:

  • 专注你的文字内容而不是排版样式；

  • 轻松的导出 HTML、PDF 和本身的 .md 文件；

  • 纯文本内容，兼容所有的文本编辑器与字处理软件；

  • 可读，直观，适合所有人的写作语言。

这里介绍一些常用的markdown语法:
  1. 标题：文章的标题用#来标记，并建议在#后面加一个空格。

示例: 

# 一级标题  
## 二级标题  
### 三级标题  
#### 以此类推，总共六级标题。

效果展示:

  1. 字体:

    • 加粗
      要加粗的文字左右分别用两个*包起来；
    • 斜体
      要倾斜的文字左右分别用一个*包起来；
    • 斜体加粗
      要倾斜和加粗的文字左右分别用三个*号包起来;
    • 删除线
      要加删除线的文字左右分别用两个~~号包起来;

示例:

**这是加粗的文字**  
 *这是倾斜的文字*  
***这是斜体加粗的文字***  
~~这是加删除线的文字~~

效果展示:

  1. 区块引用:在引用的内容前加>即可。

示例:

>这是引用的内容  
>>这是引用的内容
>>>>>>>这是引用的内容

效果展示:

  1. 列表:
    • 无序列表:使用*或+或-表示无序列表。

示例:

* 无序列表1  
+ 无序列表2  
- 无序列表3

效果展示:

  • 有序列表:有序列表则使用数字接着一个英文句点。

示例:

效果展示:

  1. 分割线:你可以在一行中用三个以上的星号、减号、底线来建立一个分隔线,

    行内不能有其他东西。你也可以在星号或是减号中间插入空格。

示例:

* * *   
- - - -   
_ _ _ _ _ _

效果展示:

  1. 图片和链接:

示例：
