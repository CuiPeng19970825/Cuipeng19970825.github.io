一、线性结构—单链表

发表于 2019-05-11 | 更新于 2019-08-23 | 分类于 数据结构与算法

单链表学习整理

单链表

通过一个链子把多个结点（元素）连接起来，由数据和地址组成的一个元素，节点本身必须有一个

地址值（就是下一个元素的地址值）链表的每个结点中只包含一个指针域，叫做单链表（即构成链

表的每个节点只有一个指向直接后继结点的指针，且尾节点的指针为null）。

优点：

  1. 链表是很常用的一种数据结构，不需要初始化容量，可以任意加减元素；

  2. 添加或者删除元素时只需要改变前后两个元素结点的指针域指向地址即可，所以添加，

删除很快；

缺点：

  1. 因为含有大量的指针域，占用空间较大；

  2. 查找元素需要遍历链表来查找，非常耗时。

适用场景：

数据量较小，需要频繁增加，删除操作的场景。

基类的定义

  • 定义一个节点类Node

  • append()方法,为当前节点追加节点

  • Next()方法，获取下一个节点

  • getData()方法，获取节点中的数据

  • isLast()方法，判断当前节点是否是最后一个节点

  • remove()方法，删除下一个节点

  • show()方法，显示所有节点信息

  • insert()方法，插入一个节点

代码实现

package demoNode;
public class Node {
       //数据
       private int data;
       //下一个节点
       private Node next;

       //构造方法
       public Node(int data){
           this.data=data;
       }

       //为当前节点追加节点
       public Node append(Node node){
           //当前节点
           Node currentNode=this;
           //循环查找
           while(true){
               //取出下一个节点
               Node nextNode=currentNode.next;
               //如果下一个节点为空，当前节点已经是最后一个节点
               if(nextNode==null){
                   break;
               }
               //赋值给当前节点
               currentNode=nextNode;
           }
           //把需要追加的节点追加到找到的当前节点的下一个节点
           currentNode.next=node;
           return this;
       }

       //获取下一个节点
       public Node Next(){
           return this.next;
       }

       //获取节点中的数据
       public int getData(){
           return this.data;
       }

       //判断当前节点是否是最后一个节点
       public boolean isLast(){
           return next==null;
       }

       //删除下一个节点
       public void remove(){
           //取出下一个节点
           Node newNext=next.next;
           //把下下个节点设置为当前节点的下一个节点
           this.next=newNext;

       }

       //显示所有节点信息
       public void show(){
           Node currentNode=this;
           while(true){
               System.out.print(currentNode.data+" ");
               //取出下一个节点
               currentNode=currentNode.next;
               if(currentNode==null){
                   break;
               }
           }
           System.out.println();
       }

       //插入一个节点
       public void insert(Node newNode){
           Node currentNode=this;
           newNode.next=currentNode.next;
           currentNode.next=newNode;
       }

   }

测试代码

 package demoNode;
 public class TestNode {
    public static void main(String[] args) {
        //创建节点对象
        Node n1=new Node(1);
        Node n2=new Node(2);
        Node n3=new Node(3);
        //用追加节点方法将相应的节点连成单链表
        n1.append(n2).append(n3).append(new Node(4));
        //取出下一个节点中的数据
        System.out.println(n1.Next().Next().getData());
        System.out.println(n1.isLast());
        //显示所有节点信息
        n1.show();
        //删除某个节点的下一个节点
        n1.remove();
        n1.show();
        //插入节点
        n1.insert(n2);
        //追加新节点
        n1.append(new Node(5));
        n1.show();
        System.out.println(n2.getData());
    }
}

队列学习整理

队列

队列(queue)是一种特殊的线性表 ，它只允许在表的前端（front）进行删除操作，而在表的后端

（rear）进行插入操作。进行插入操作的端称为队尾，进行删除操作的端称为队头。队列是按照“先

进先出”或“后进后出”的原则组织数据的。队列中没有元素时，称为空队列。

优点：提供先进先出的存取方式

缺点：存取其他项都很慢

基类的定义

  • 定义一个队列类Queue

  • add()方法，入队

  • poll()方法，出队

  • isEmpty()方法，判断队列是否为空

代码实现

package demoQueue;
public class Queue {
       int[] arr;
       public Queue(){
           arr=new int[0];
       }

       //入队
       public void add(int element){
           // 新创建一个数组
           int[] newarr = new int[arr.length + 1];
           // 将原数组中的元素复制到新数组中
           for (int i = 0; i < arr.length; i++) {
               newarr[i] = arr[i];
           }
           // 将新增加的元素放入新数组中
           newarr[arr.length] = element;
           // 新数组替换原数组
           arr = newarr;
       }

       //出队
       public int poll(){
           int element=arr[0];
           //新创建一个数组
           int[] newarr=new int[arr.length-1];
           //将未出栈的元素复制到新数组中
           for(int i=0;i<newarr.length;i++){
               newarr[i]=arr[i+1];
           }
           //新数组替换原数组
           arr=newarr;
           //返回出队的元素
           return element;
       }

       //判断队列是否为空
       public boolean isEmpty(){
           return arr.length==0;
       }
   }

栈学习整理

栈

栈是一种只允许在一端进行插入或删除的线性表，也就是说先进后出。栈的操作端通常被称为栈顶，

另一端被称为栈底，栈的插入操作称为压栈（push），栈删除操作称为出栈（pop）。压栈是把新元

素放到栈顶元素的上面，使之成为新的栈顶元素；出栈则是把栈顶元素删除掉，使其相邻的元素成

为新的栈顶元素。

优点：提供后进先出的存取方式

缺点：存取其他项很慢

基类的定义

  • 定义一个栈类Stack

  • push()方法，压入元素

  • pop()方法，取出栈顶元素

  • size()方法，显示栈的大小

  • show()方法，查看栈顶元素

  • isEmpty()方法，判断栈是否为空

代码实现

 package demoStack;
 public class Stack {
        private int[] arr;
        public Stack(){
            arr=new int[0];
        }

        //压入元素
        public void push(int element){
            // 新创建一个数组
            int[] newarr = new int[arr.length + 1];
            // 将原数组中的元素复制到新数组中
            for (int i = 0; i < arr.length; i++) {
                newarr[i] = arr[i];
            }
            // 将新增加的元素放入新数组中
            newarr[arr.length] = element;
            // 新数组替换原数组
            arr = newarr;
        }

        //取出栈顶元素
        public void pop(){
            //新创建一个数组
            int[] newarr=new int[arr.length-1];
            //将原数组中的元素复制到新数组中
            for(int i=0;i<newarr.length;i++){
                newarr[i]=arr[i];
            }
            //新数组替换原数组
            arr=newarr; 
        }

        //显示栈的大小
        public int size(){
            return arr.length;
        }

        //查看栈顶元素
        public void show(){
            System.out.println(arr[arr.length-1]);
        }

        //判断栈是否为空
        public boolean isEmpty(){
            return arr.length==0;
        }
}

一、线性结构—数组

数组学习整理

数组

数据结构中最基本的一个结构就是线性结构，而线性结构又分为连续存储结构和离散存储结构，

所谓的连续存储结构其实就是数组。

优点 ：

  1. 按照索引查询元素速度快

  2. 按照索引遍历数组方便

缺点：

  1. 数组的大小固定后就不能扩容了

  2. 数组只能存储一种类型的数据

  3. 添加，删除的操作慢，因为要移动其他的元素

适用场景：

频繁查询，对存储空间要求不大，很少增加和删除的情况

基类的定义

  • 定义一个数组类Arr

  • size()方法，获取数组长度

  • add()方法，向数组末尾增加一个元素

  • show()方法，显示数组中的所有元素

  • delete()方法，删除数组中某个位置的元素

  • insert()方法，向数组中插入一个元素

  • get()方法，获取数组中目标位置的元素

  • set()方法，替换数组中目标位置的元素值

  • search()线性查找方法

  • BinarySearch()二分法查找，条件：查找的数组有序

代码实现

package demoArray;
import java.util.Arrays;
public class Arr {
    private int[] arr;

    public Arr (){
        arr=new int[0];

    }

    //获取数组长度
    public int size(){
        return arr.length;
    }

    //向数组末尾增加一个元素
    public void add(int element){
        //新创建一个数组
        int[] newarr=new int[arr.length+1];
        //将原数组中的元素复制到新数组中
        for(int i=0;i<arr.length;i++){
            newarr[i]=arr[i];
        }
        //将新增加的元素放入新数组中
        newarr[arr.length]=element;
        //新数组替换原数组
        arr=newarr;
    }

    //显示数组中的所有元素
    public void show(){
        System.out.println(Arrays.toString(arr));
    }

    //删除数组中某个位置的元素
    public void delete(int index){
        //创建一个新数组
        int[] newarr=new int[arr.length-1];
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        for(int i=0;i<newarr.length;i++){
            //将删除元素前的元素复制到新数组中
            if(i<index){
            newarr[i]=arr[i];
            //将删除元素后的元素复制到新数组中
            }else{
                newarr[i]=arr[i+1];
            }
        }
        //新数组替换原数组
        arr=newarr;
    }

    //向数组中插入一个元素
    public void insert(int index,int element){
        //创建一个新数组
        int[] newarr=new int[arr.length+1];
        //判断下标是否越界
        if(index<0||index>newarr.length-1){
            throw new RuntimeException("下标越界");
        }
        for(int i=0;i<arr.length;i++){
            //将插入元素前的元素复制到新数组
            if(i<index){
                newarr[i]=arr[i];
            //将插入元素后的元素复制到新数组
            }else{
                newarr[i+1]=arr[i];
            }
        }
        //将插入元素放入到新数组中
        newarr[index]=element;
        //新数组替换原数组
        arr=newarr;
    }

    //获取数组中目标位置元素
    public int get(int index){
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        return arr[index];
    }

    //替换数组中目标位置元素的值
    public void set(int index,int element){
        //判断目标位置是否越界
        if(index<0||index>arr.length-1){
            throw new RuntimeException("下标越界");
        }
        arr[index]=element;
    }

    //线性查找
    public int search(int target) {
        for(int i=0;i<arr.length;i++){
            if(arr[i]==target){
                return i;
            }
        }
        return -1;
    }

    //二分法查找
    public int BinarySearch(int target){
        // 记录开始位置
        int begin = 0;
        // 记录结束位置
        int end = arr.length - 1;
        // 记录中间的位置
        int mid = (begin + end) / 2;
        // 循环查找
        while (true) {
            //判断什么情况下没有目标元素
            if(begin>=end){
                return -1;
            }
            // 判断中间的这个元素是否是目标元素
            if (arr[mid] == target) {
                return mid;
            } else {
                if (arr[mid] > target) {
                    end = mid - 1;
                } else {
                    begin = mid + 1;
                }
                mid = (begin + end) / 2;
            }
        }
    }
 }

git学习

发表于 2019-04-30 | 更新于 2019-08-23 | 分类于 工具

前言：git的学习记录，学习参考网站Git教程-廖雪峰的官方网站

一、

git定义：分布式版本控制系统

git作用：记录、管理文件的每一次改动，支持协作编辑。

  1. 版本控制系统：记录软件更新内容的系统

  2. 集中式：版本库是集中存放在中央服务器中的，工作时需要从中央服务器中取得最新的版本，更

    新版本内容后再把最新的版本提交到中央服务器，举例：中央服务器就好比是一个图书馆，你要

    改一本书，必须先从图书馆借出来，然后回到家自己改，改完了，再放回图书馆。

  3. 分布式：分布式版本控制系统没有“中央服务器”，每个人的电脑上都是一个完整的版本库，协作

    时只需将更新的内容推送给对方即可。

  4. 集中式vs分布式：集中式版本控制系统最大的毛病就是必须联网才能工作，而分布式版本控制系

    统没有中央服务器，每个人电脑里都有完整的版本库，不需要联网，同时提高了安全性问题，当

    某一人电脑坏了的时候只需要从其他人电脑里把版本库复制一份即可，而集中式版本控制系统若

    中央服务器出问题会导致所有人都无法工作。

二、

同类工具：

  1. CVS最早的开源而且免费的集中式版本控制系统，直到现在还有不少人在用。由于CVS自身设

    计的问题，会造成提交文件不完整，版本库莫名其妙损坏的情况。同样是开源而且免费的SVN

    修正了CVS的一些稳定性问题，是目前用得最多的集中式版本库控制系统。

  2. 除了免费的外，还有收费的集中式版本控制系统，比如IBM的ClearCase（以前是Rational公司

    的，被IBM收购了），特点是安装比Windows还大，运行比蜗牛还慢。微软自己也有一个集中式版

    本控制系统叫VSS，集成在Visual Studio中。

  3. 分布式版本控制系统除了Git以及促使Git诞生的BitKeeper外，还有类似Git的Mercurial和Ba

    zaar等。这些分布式版本控制系统各有特点，但最快、最简单也最流行的依然是Git！

三、

安装

  1. 安装教程连接

    https://git-scm.com/book/zh/v2/%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git

语法和用法：

  1. 创建版本库

    mkdir blogimages 版本库又名仓库，英文名respository,blogimages为仓库名

    cd blogimages 切换到blogimages仓库

    pwd 显示当前目录位置

  2. 初始化仓库

    git init 通过该命令把这个目录变成git可以管理的仓库

  3. 将更新的内容放到该仓库中 这里在该仓库中添加了一个img文件夹

  4. 将文件添加到仓库

  5. 将文件提交到仓库

    git commit -m “add two images” -m后面输入的是本次提交的说明

  6. 查看仓库当前的状态

    git status 该命令可以让我们时刻掌握仓库当的状态

  7. 查看修改前后的差别

    git diff 查看修改前后的difference

  8. 查看提交历史

    git log git log命令显示从最近到最远的提交日志

  9. 从当前版本回退到上一个版本

    git reset -hard Head^

    Head表示当前版本，Head^表示上一个版本，Head^^表示上上一个版本，Head~100表示往上100个版本

  1. 查看命令历史

    git reflog git提供了一个命令git reflog用来记录你的每一次命令

  2. 工作区和暂存区

    • 在你电脑里能看到的目录，即上面所建的blogimages为工作区

    • 工作区中的.git不算工作区，而是git的版本库

    • 版本库中的stage(或者叫index)为暂存区，master为git自动给我们创建的第一个分支

    • 前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的：

      第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区；

      第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。

  3. 查看工作区和版本库里最新版本的区别

    git diff Head – img

  4. 丢弃工作区的修改

    git checkout – img

注释：

一种是img自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的
状态；
一种是img已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后
的状态,总之，就是让这个文件回到最近一次git commit或git add时的状态。

  1. 把暂存区的修改撤销掉，重新放回工作区

    git reset Head img

    由此可知道git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区

  2. 删除文件

    rm 文件名

事务

发表于 2019-01-03 | 更新于 2019-10-08 | 分类于 数据库

一、什么是数据库事务

1、定义:数据库事务是构成单一逻辑工作单元的操作集合

2、关于定义的解释：

  1. 数据库事务可以包含一个或多个数据库操作,但这些操作构成一个逻辑上的整体。
  2. 构成逻辑整体的这些数据库操作,要么全部执行成功,要么全部不执行。
  3. 构成事务的所有操作,要么全都对数据库产生影响,要么全都不产生影响,即不管事务是否执行成
    功,数据库总能保持一致性状态。
  4. 以上即使在数据库出现故障以及并发事务存在的情况下依然成立。

二、事务控制语句：

  1. BEGIN 或 START TRANSACTION 显式地开启一个事务；

  2. COMMIT 也可以使用 COMMIT WORK，不过二者是等价的。COMMIT 会提交事务，并使已对数据
    库进行的所有修改成为永久性的；

  3. ROLLBACK 也可以使用 ROLLBACK WORK，不过二者是等价的。回滚会结束用户的事务，并撤销
    正在进行的所有未提交的修改；

  4. SAVEPOINT identifier，SAVEPOINT 允许在事务中创建一个保存点，一个事务中可以有多个
    SAVEPOINT；

  5. RELEASE SAVEPOINT identifier 删除一个事务的保存点，当没有指定的保存点时，执行该语
    句会抛出一个异常；

  6. ROLLBACK TO identifier 把事务回滚到标记点；

  7. SET TRANSACTION 用来设置事务的隔离级别。InnoDB 存储引擎提供事务的隔离级别有READ
    UNCOMMITTED、READ COMMITTED、REPEATABLE READ 和 SERIALIZABLE。

MYSQL 事务处理主要有两种方法：

1、用 BEGIN, ROLLBACK, COMMIT来实现
BEGIN 开始一个事务
ROLLBACK 事务回滚
COMMIT 事务确认
2、直接用 SET 来改变 MySQL 的自动提交模式:
SET AUTOCOMMIT=0 禁止自动提交
SET AUTOCOMMIT=1 开启自动提交

三、事务的ACID特性

一般来说，事务是必须满足4个条件（ACID）：原子性（Atomicity，或称不可分割性）、一致性（Consistency）、隔离性（Isolation，又称独立性）、持久性（Durability）。

1.原子性：一个事务（transaction）中的所有操作，要么全部完成，要么全部不完成，不会结束在中间某个环节。事务在执行过程中发生错误，会被回滚（Rollback）到事务开始前的状态，就像这个事务从来没有执行过一样。

2.一致性：在事务开始之前和事务结束以后，数据库的完整性没有被破坏。比如转账前后两个账户的金额总和应该保持不变。

3.隔离性：数据库允许多个并发事务同时对其数据进行读写和修改的能力，并发执行的事务不会相互影响,其对数据库的影响和它们串行执行时一样。比如多个用户同时往一个账户转账,最后账户的结果应该和他们按先后次序转账的结果一样。事务隔离分为不同级别，包括读未提交（Read uncommitted）、读提交（read committed）、可重复读（repeatable read）和串行化
（Serializable）。

4.持久性：事务处理结束后，对数据的修改就是永久的，即便系统故障也不会丢失。

在 MySQL 命令行的默认设置下，事务都是自动提交的，即执行 SQL 语句后就会马上执行COMMIT
操作。因此要显式地开启一个事务务须使用命令 BEGIN 或 START TRANSACTION，或者执行命令 SET AUTOCOMMIT=0，用来禁止使用当前会话的自动提交。

隔离级别分析

  1. 读未提交（Read uncommitted）——脏读：是指一个事务读取了另一个事务未提交的数据

    在事务1对A的处理过程中,事务2读取了A的值,但之后事务1回滚,导致事务2读取的A是未提交的脏数据。

  2. 读提交(read committed)——不可重复读:是指一个事务对同一数据的读取结果前后不一致。

    由于事务2对A的已提交修改,事务1前后两次读取的结果不一致。

  3. 可重复读(repeatable read)——幻读
    是指事务读取某个范围的数据时，因为其他事务的操作导致前后两次读取的结果不一致。幻读和不可重复读的区别在于,不可重复读是针对确定的某一行数据而言,而幻读是针对不确定的多行数据。因而幻读通常出现在带有查询条件的范围查询中,比如下面这种情况:

    事务1查询A<5的数据,由于事务2插入了一条A=4的数据,导致事务1两次查询得到的结果不一样

  4. 串行化(Serializable)
    当一个事务提交之后，另一个事务才可以执行，串行化可以避免所有可能出现的并发异常,但是会极大的降低系统的并发处理能力。

级别由低到高依次为Readuncommitted、Readcommitted、Repeatableread、Serializable。

其他常见的数据库并发异常

  1. 脏写：是指事务回滚了其他事务对数据项的已提交修改,比如下面这种情况

    在事务1对数据A的回滚,导致事务2对A的已提交修改也被回滚了。

  2. 丢失更新
    丢失更新是指事务覆盖了其他事务对数据的已提交修改,导致这些修改好像丢失了一样。

    事务1和事务2读取A的值都为10,事务2先将A加上10并提交修改,之后事务2将A减少10并提交修改,A的值最后为0,导致事务2对A的修改好像丢失了一样

SQL的四种连接查询

发表于 2019-01-03 | 更新于 2019-10-08 | 分类于 数据库

一、内连接

1.内连接（inner join 或 join）
内连接是等值连接，它使用“=、>、<、<>”等运算符根据每个表共有的列的值匹配两个表中的行
查询语句：
select * from T_Person inner join T_Card on T_Person.CardId = T_Card.CardId

二、外连接

2.左连接（left join 或 left outer join）
左连接又称左向外连接，查询的结果集包括SQL语句中左表的所有行，右表中匹配的行。如果
左表的某行在右表中没有匹配行，则用空值表示。
查询语句：
select * from T_Person left join T_Card on T_Person.CardId = T_Card.CardId

3.右连接（right join 或 right outer join）
右连接也称右向外连接，查询的结果集包括SQL语句中右表的所有行，左表中匹配的行。如果
右表的某行在左表中没有匹配的行，则用空值表示
查询语句：
select * from T_Person right join T_Card on T_Person.CardId = T_Card.CardId

4.完全外连接（full join 或 full outer join）
完全外连接，查询的结果集包括SQL语句中左表和右表的所有行。如果某行在另一个表中没有
匹配行时，则用空值表示。
查询语句：
select * from T_Person full join T_Card on T_Person.CardId = T_Card.CardId
