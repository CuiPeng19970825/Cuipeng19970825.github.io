Java中类的实例化顺序

发表于 2018-03-05 | 更新于 2019-09-13 | 分类于 Java

总结

  1. 先静态、先父后子。 先静态：父静态 > 子静态 。
  2. 优先级：父类 > 子类 ， 静态代码块 > 非静态代码块 > 构造函数。

例题分析

例一：

    package Servlet.day0811.am;
    public class StaticTest {
    /*1、静态代码块先加载，当new一个新对象时调用构造方法且该类的非静态代码块要执行一次
    注意：(静态代码块与非静态代码块的区别，静态代码块只在第一次new的时候执行一次,
        而非静态代码块在每次new的时候都要执行一次)*/
    static StaticTest st=new StaticTest();  
    //5、按顺序执行静态代码块 
    static{
        System.out.println("静态代码块");
    }
    //3、构造方法执行
    public StaticTest(){
        System.out.println("构造器");
        System.out.println(st);
        m1();
    } 
    //4、m1方法被调用  7、m1方法再次被调用
    public static void m1(){
        System.out.println("m1静态方法");
        //执行到顺序7时，构造方法已经完全执行完，即st对象被实例化
        System.out.println(st);
    }
    {
        //2、此非静态代码块执行(因为非静态构造方法的优先级高于构造方法)
        System.out.println("普通代码块");
    }
    //6、按顺序执行静态代码块
    public static void main(String[] args) {
        System.out.println("main开始");
        StaticTest.m1();
        //8、按顺序执行
        System.out.println("main结束");
    }
}

输出结果

普通代码块 （先加载普通代码块）
构造器 （再加载构造器）
null （输出构造器中的st，此时并未加载静态属性，所以输出null）
m1静态方法 （构造器中的m1方法，输出m1静态方法）
null （m1方法中的st，此时st仍然未被加载）
静态代码块 （加载静态代码块）
main开始 （main方法开始，st在这时候被加载（分配内存））
m1静态方法 （调用静态方法m1）
Servlet.day0811.am.StaticTest@d1f24bb （st已经被加载过了，所以为对象对应的地址）
main结束 （main方法结束）

例二：

    public class Dervied extends Base{
    private String name="dervied";
    public Dervied(){
        //4、执行构造方法，此时new了一个父类的对象，调用父类构造方法
        System.out.println(new Base().i);//8、按顺序继续执行输出i的值
        tellName();
        printName();
    }
    static{
        //2、子类静态代码块再执行
        System.out.println("子类静态代码块");
    }
    //9、子类构造函数调用tellName和printName函数
    public void tellName(){
        System.out.println("Dervied tell name : "+name);
    } 
    public void printName(){
        System.out.println("Dervied print name : "+name);
    }
    public static void main(String[] args) {
        //3、子类静态代码块按顺序执行，此时调用子类的构造方法
        new Dervied();
    }
}
    class Base{
        public int i=99;
        private String name="111";
        static{
            //1、父类静态代码块先执行
            System.out.println("父类静态代码块");
        }
        {
            //5、当new一个父类对象时非静态代码块就要执行一次
            System.out.println("父类非静态代码块");
        }
        //6、执行父类构造函数
        public Base(){
            tellName();
            printName();
        }
        //7、父类构造函数调用tellName和printName函数
        public void tellName(){
            System.out.println("Base tell name : "+name);
        } 
        public void printName(){
            System.out.println("Base print name : "+name);
        }
    }

输出结果

父类静态代码块
子类静态代码块
父类非静态代码块
Dervied tell name : null
Dervied print name : null
父类非静态代码块
Base tell name : 111
Base print name : 111
99
Dervied tell name : dervied
Dervied print name : dervied
